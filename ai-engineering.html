AI Engineering Method

把 AI 从代码生成器变成可控的工程协作者

通过结构化上下文、阶段状态、交付物和门禁，约束 AI 的工程行为——让 AI 负责加速执行，让人负责目标、边界和最终责任，使每一次交付可追踪、可审查、可复用。

Core Principle

核心观点

AI 可以判断错，但工程流程不能失控。

AI 的职责

负责加速搜索、理解、生成、审查和文档整理。在明确边界内提高执行效率，不替代人对方向和风险的判断。

搜索 理解 生成 审查 文档整理

人的职责

负责目标、边界、架构、风险、验收和最终责任。决定做什么、不做什么、做到什么程度，并对结果承担责任。

目标 边界 架构 风险 验收 最终责任

工程控制

通过结构化上下文、阶段状态、交付物和门禁，降低模型随机性，使 AI 的输出可追踪、可审查、可交付。

结构化上下文 阶段状态 交付物 门禁

Domain Entry

进入陌生领域的方法

面对陌生业务领域，先收集输入、提取领域上下文，再形成可执行的工程产出物。

输入源

  • 业务说明
  • 接口文档
  • 遗留源码
  • 数据库结构
  • 设备协议
  • 现场约束
  • 运行日志
  • 已知问题

输出物

  • 术语表
  • 领域对象
  • 系统边界
  • 调用拓扑
  • 接口清单
  • 状态机
  • 风险矩阵
  • 实施计划

不是让 AI 直接写代码，而是先建立足够可靠的领域上下文。

Multi-Model Workflow

多模型协作流程

规划、实现、审查、修复和验收由不同阶段承担，每个阶段有明确的输入、输出和门禁。

01
Plan

规划

02
Build

实现

03
Review

独立审查

04
Fix

问题修复

05
Final Review

最终验收

Plan
独立模型输出范围、架构、风险、步骤和验收标准。
Build
执行模型按计划开发，并记录实际变更。
Review
不同模型只做独立审查，不默认相信实现结果。
Fix
根据问题清单逐项修复并复测。
Final Review
对照需求、计划、Git 变更和测试证据做最终结论。

Risk Classification

风险分级

按变更影响范围将工程任务分为三档，每档对应不同的控制强度和验收要求。

低风险

文案、样式、普通页面、非关键脚本。

允许 AI 快速迭代

中风险

API、数据库、业务状态、外部系统集成、部署配置。

要求测试 要求独立审查

高风险

设备控制、PLC 写入、并发、状态机、权限、资金 / 订单 / 不可逆操作。

明确状态 幂等 锁 超时 重试边界 审计日志 人工验收 失败恢复

Definition of Done

完成定义

完成不是 AI 声称完成，而是存在可验证的交付证据。

不等于完成

  • 页面打开不等于完成。
  • API 返回 200 不等于业务正确。
  • Mock 通过不等于真实设备可用。
  • 单节点成功不等于全网发布成功。
  • AI 声称完成不等于存在验收证据。

完成必须包含

  • 可运行代码
  • 测试结果
  • 真实或明确标记的模拟验证
  • 部署说明
  • 回滚路径
  • 日志和诊断
  • 未完成项列表
  • 可接手文档

Scale / Engine

如何形成 Scale / Engine

把一次性交付继续抽象为可复用的工程资产，让后续人员和 AI Agent 按统一路径接手。

项目问题 提取重复模式 分离业务差异与公共内核 固化为 Adapter / Skill / Template / Checklist 加入测试和验收门禁 让后续人员与 Agent 使用

主要实例：SimpleHmi_WEILI

SimpleHmi_WEILI 是这一抽象过程的主要实例。它从数字月台和机器人产线两个真实项目中提取工业 HMI / 控制系统的重复工程模式，分离业务差异与公共内核，固化为规范文档、Adapter、检查清单、验收门禁和自修复流程，使后续人员和 AI Agent 能够按统一路径生成可运行、可部署、可验收的工业系统。

Transferability

迁移能力

该方法可迁移到以下需要可靠交付的场景：

企业内部工具

内部系统、管理后台、运营工具的快速构建与交付。

智能硬件应用

设备接入、状态管理、控制逻辑与人机界面的工程化实现。

AI 助手和 Agent

Agent 工作流、上下文工程、结构化输出与多模型协作。

业务自动化

流程编排、任务调度、异常恢复与可审计的自动化链路。

遗留系统改造

代码阅读、调用拓扑、状态机重构与渐进式迁移。

数据采集和控制台

设备数据采集、实时监控、报警规则与运维控制台。

多系统 API 集成

接口契约、数据映射、幂等处理与集成测试。

垂直行业应用

需要可靠交付、存在验收标准和不可逆操作约束的行业场景。

迁移的前提是目标场景存在明确的交付标准和可验证的工程边界，而非无限扩大适用范围。
