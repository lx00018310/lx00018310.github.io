重点项目

数字月台工业边缘中控与多 HMI 协同系统

核心开发 / 系统集成｜2026.03 - 至今

项目背景：面向工厂自动装车与数字月台场景，建设一套由公共工控机、12 台触摸屏 HMI、PLC、调度系统、人脸识别终端和局域网发布系统组成的工业边缘中控系统。系统需要支持月台作业展示、人员核验、设备控制、报警处理、状态推送、版本统一发布和现场部署恢复。

技术架构：React + TypeScript + Vite / Node.js + Fastify + WebSocket / PostgreSQL / SLMP 3E PLC / Python + 海康 ISUP 5.0 / HMI Agent / Windows 工控机 / Nginx / NSSM / PowerShell

核心工作：

我在该项目中承担核心开发与系统集成职责，覆盖以下六个主要方向：

  • HMI 前端与角色分层：使用 React、TypeScript、Vite 构建竖屏全屏 HMI 应用，搭建 12 台触摸屏前端，完成登录确认、作业信息、设备控制、报警、管理调试等核心页面；针对操作工、主管、维护人员设计不同界面层级与权限显隐，A/B 机型差异处理，降低一线人员操作复杂度。
  • Node.js / Fastify / WebSocket 中控后端：使用 Node.js、Fastify、WebSocket 与 PostgreSQL 建设中控后端，完成月台状态存储、任务状态流转、事件总线、实时推送、关键运行态持久化与服务重启恢复；建立调度系统 Dispatch Adapter，支持 mock / real 模式切换，对调度入站接口、DTO 校验、结构化日志和异常链路进行收口。
  • PLC 点位与网关链路：接入真实 SLMP 3E PLC 网关，完成 12 个月台 PLC、配零出口 PLC、工位、扫码枪、人脸设备和工控机等 IP 与点位配置；设计 PLC 独占排队锁机制，实现后台轮询、来源标记和安全联锁，防止并发写冲突；设计 HMI 客户端 IP 绑定与校验机制，限制非授权终端操作。
  • 人脸 ISUP 服务联调：基于 Python 与海康 ISUP 5.0 SDK 建设独立人脸边缘服务（进程隔离防 SDK 崩溃株连），打通设备注册、状态监听、报警事件解析、人员下发、人脸图片下发、照片回采与 HMI 工位绑定；优化人脸识别到 HMI 前端的事件链路，解决人脸事件解析、设备序列号映射、图片通道阻塞、WebSocket 推送和前端状态无响应等问题。
  • HMI Agent / 部署控制 / 版本回滚：设计部署控制器与 HMI 常驻代理（PowerShell），实现 12 台终端身份认证、心跳检测、版本管理、完整性校验、统一激活、失败回滚与离线恢复；设计 .wrelease 不可变发布包和目标版本自动收敛机制，支持局域网全网发布、版本校验、前端 Mock UI 禁用校验和异常包拦截，任一节点失败全网回滚，中断时间 < 90 秒。
  • Windows 服务化、自愈脚本、联调文档：推进 Windows 原生绿色服务化部署，使用 PowerShell、NSSM 和健康检查脚本完成一键安装、两级服务守护（NSSM 5秒拉起 + 控制器主动巡检）、崩溃重启、死锁自愈和异常文件修复；设计 HMI Bootstrap 兜底机制；坚持生产链路与调试链路隔离，关键配置缺失时阻断启动或发版；输出完整的联调文档、SOP和异常排查手册。

技术架构：Java / Spring Boot / Vue / MySQL / Android Kiosk / Android 机器人 SDK / PLC / TCP 喷码器 / USB 读卡器 / 本地 HTTP 服务 / Windows 工控机

  • 基于 Spring Boot、Vue 和 MySQL 收口刷卡选味、生产状态、组垛、喷码、机器人调度和异常复位流程，建设操作员控制面板和设备状态监控能力。
  • 设计业务状态机，将自助刷卡、口味选择、投桶、PLC 信号确认、喷码、机器人取餐、送餐、返程等环节串联为可追踪流程。
  • 封装 PLC、机器人、喷码器和读卡器接入层，处理多设备协议差异和现场接口不稳定问题。

  • 开发 Android 机器人 Kiosk 与本地 HTTP 控制服务，实现路线配置、按桌送餐、人工返程、正式回充、取消任务和解锁复位。

  • 实现机器人精密脉冲点动、定位恢复、地图加载、急停复检、电量检查、避障熔断、实时位姿校验和环境预检。
  • 建设硬件模拟调试面板与机器人调试 API，使 PLC 点位、机器人路线、刷卡、喷码等链路可在联调阶段分层验证。
  • 推进生产配置收敛，区分 dev / prod 环境，关键 PLC、喷码器、机器人回调、数据库、Redis 等配置从环境变量或统一配置文件读取。

  • 增加终端心跳、运行时配置、前端联调启动脚本、后端生产启动脚本和一键打开工控机所有前端脚本，提高现场部署效率。
