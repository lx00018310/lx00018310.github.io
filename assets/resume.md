### 1. SimpleHmi_WEILI：AI-first 工程规范

**维护者 / 工程抽象｜2026.04 - 至今**

开源仓库：https://github.com/lx00018310/SimpleHmi_WEILI
项目页：https://lx00018310.github.io/simplehmi-weili.html

**项目背景：**

从数字月台和机器人产线两个真实复杂项目中提取工程经验，沉淀为一套 AI-first 工业 HMI / 控制系统工程规范。它面向不懂代码但能描述现场需求的人，约束 AI 如何读取上下文、选择技术栈、规划系统、接入设备、执行测试、部署和提交验收证据，生成可运行、可部署、可验收的工业现场系统。

**技术架构：**

规范文档（README、START_HERE、AGENTS、SIMPLE_HMI、ENGINES、WEILI_LADDER、docs 索引）+ templates 模板 + checklists 验收门禁 + examples/minimal-hmi-system 最小可运行样板（React + TypeScript + Vite + Fastify + Node.js）

**核心工作：**

* 设计 AI Agent 的上下文入口与读取顺序（START_HERE_FOR_BEGINNERS → AGENTS → SIMPLE_HMI → ENGINES → WEILI_LADDER → docs/00_INDEX）。
* 把"真实现场优先、主链路优先、默认禁止 Mock、完成必须有证据"固化为可检查、可拦截的规则。
* 约束 AI 默认使用 React、TypeScript、Vite、Node.js、Fastify 等轻量技术栈，能不做就不做，能用现有能力就不用新框架。
* 提供需求澄清、架构规划、接口契约、设备接入、配置、验收等模板，以及需求、架构、契约、设备、去 Mock、部署、现场验收、AI 完成证据等门禁清单。
* 提供最小可运行样板 examples/minimal-hmi-system，展示一个合格的最小工业 HMI 系统应该如何组织。
* 把"未理解需求就开始写代码、Mock 冒充真实、只验证前端页面、忽略部署、修复一个问题引入另一个问题"等失败模式前置成可检查、可拦截的规则。

**AI 如何参与：**

AI Agent 是这套规范的执行对象。AI 按规范读取上下文、先规划再编码、生成前后端与设备 Adapter、出错时按自修复规范搜索定位修改复测，最终输出验收证据。

**可复用沉淀：**

AI-first 工程规范、AI Agent 上下文与读取顺序、技术栈边界约束、No Mock 生产门禁、验收门禁清单、自修复流程、最小可运行样板。

**项目状态：**

已完成规范主体、模板、检查清单和最小可运行样板，已公开发布并持续迭代。
