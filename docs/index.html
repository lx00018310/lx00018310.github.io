Featured AI Engineering Project

把项目经验变成 AI 可执行的工程规范

SimpleHmi_WEILI

SimpleHmi_WEILI 是一套 AI-first 工业 HMI / 控制系统工程规范。它把自然语言现场需求转化为规划、架构、代码、设备适配、部署脚本和验收证据，并通过 No Mock、配置校验、测试门禁、自修复和多模型审查约束 AI 的工程行为。

让普通开发者借助 AI，生成可运行、可部署、可验收的工业系统。

  • 将隐性经验结构化
  • 为 AI 建立上下文和读取顺序
  • 将生产边界写成自动门禁
  • 让后续人员获得可执行的接手路径
查看项目方法 打开 GitHub 仓库

来自真实项目的抽象

SimpleHmi_WEILI 不是凭空设计的规范，而是从数字月台（TASK-013）和机器人产线（TASK-010）两个真实复杂项目中提取的工程经验。

它把现场联调中踩过的坑——Mock 冒充真实、配置写死、部署不可回滚、验收没有证据——前置成 AI 可读取、可搜索、可执行的规则。

形成"方法 → 开源抽象 → 真实项目验证"的完整证据链。

Proof Cases

真实复杂场景验证

工业现场不是职业边界，而是验证 AI 工程方法是否真正可靠的高约束环境。

数字月台工业边缘中控与 12 台 HMI 协同系统

场景复杂度

公共工控机、12 台 HMI、12 台 PLC、调度系统、人脸终端、WebSocket、PostgreSQL、局域网发布与回滚。

AI 的作用

辅助梳理协议与业务资料，建立系统架构、接口模型、状态机、联调计划、代码审查和部署材料。

工程难点

多终端一致性、PLC 并发写保护、外部 SDK 进程隔离、状态持久化、全网发布、失败回滚与异常恢复。

可复用沉淀

HMI Agent、部署控制器、Adapter 模式、状态机规则、生产门禁、部署和故障排查 SOP。

主要软件链路完成 本机集成验证

智能餐饮产线与送餐机器人协同控制系统

场景复杂度

读卡器、选味终端、PLC、喷码器、生产线、机器人、场馆终端、Java / Vue 遗留系统。

AI 的作用

辅助阅读遗留系统、生成调用拓扑、执行文档—前端—后端—设备一致性审查，并协助完成状态机重构与问题定位。

工程难点

机器人 API 阻塞、串口设备竞争、流程状态不可追踪、地图恢复、运行配置分散和生产调试边界混乱。

可复用沉淀

设备 Adapter、独占读卡服务、串行任务队列、状态缓存、硬件模拟面板、调试 API 和启动部署脚本。

主要流程完成 单机测试完成
