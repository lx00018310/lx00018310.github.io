AI Engineering Perspective

AI 应用工程师视角

从遗留系统到可维护架构，AI 如何参与改造，人必须负责哪些判断，最终沉淀出哪些可复用抽象。

我面对的陌生系统
  • Java / Spring Boot / Vue 遗留项目
  • PLC、读卡器、喷码器、机器人
  • 多设备异步流程
  • 分散的运行配置
  • 文档与实现不一致
AI 如何参与
  • 快速阅读仓库
  • 生成类、函数、API、数据库和设备调用拓扑
  • 对照文档、前端、后端和真实设备执行一致性审查
  • 定位阻塞链路
  • 辅助拆分状态机和设备 Adapter
  • 生成修复计划与复测清单
人必须负责的判断
  • 串口设备唯一归属
  • 机器人命令串行化
  • 导航轮询策略
  • 设备异常后的恢复路径
  • dev / prod 隔离
  • 真实设备状态的判定
最终抽象
  • 独占读卡服务
  • 串行任务队列
  • 设备 Adapter
  • 状态缓存
  • 硬件模拟面板
  • 调试 API
  • 配置和启动脚本规范

Supplement Case 02

自动化餐饮产线与送餐机器人协同控制系统

面向自动化餐饮 / 展陈式产线场景，系统串联刷卡器、选味屏、PLC、喷码器、产线状态机、机器人取餐送餐和返程，前端 Vue + 后端 Node.js。

刷卡器 选味屏 后端状态机 PLC / 喷码 机器人取餐 送餐 / 返程

读卡服务独占

避免页面或进程竞争串口 / HID 设备，读卡归属以独立服务持有，提升刷卡稳定性。

机器人 API 卡顿定位

通过串行任务、异步导航轮询、状态缓存和 Kiosk 降级隔离，解决机器人 API 高延迟导致的控制卡顿问题。

路线与地图恢复

处理路线区域合并、路径点配置、地图自动恢复、环境预检和导航异常；同一框架适配杭州母版和天津差异场景。

生产配置收敛

区分 dev / prod，PLC、喷码器、机器人回调、数据库和 Redis 统一从配置文件加载，杜绝硬编码。
