SimpleHmi_WEILI

一个面向 AI 的工业 HMI / 工控前后端统一栈规范仓库。它不是重代码框架， 而是把标准、流程、索引、模板和避坑经验写清楚，让 AI 按统一规则生成极简、可运行、可验收的工业工控系统。

工业 HMI AI 生成规范 React / TypeScript / Vite Fastify / 轻量后端 禁用生产 Mock 部署控制评估
GitHub 仓库 查看示例页面

What It Is

它解决的问题

工业现场项目最容易失控的不是页面，而是需求边界、设备状态、接口契约、部署方式、调试隔离和验收证据。 SimpleHmi_WEILI 把这些约束前置，让 AI 先按阶段读规范，再输出代码、接口、部署和验收材料。

适用对象

工业 HMI、触摸屏一体机、工位操作台、Kiosk 自助终端、公共工控机 + 多屏终端，以及 PLC、读卡器、人脸机、机器人、调度系统等外部设备集成项目。

边界声明

它不是通用后台管理模板，也不搬运任何具体业务项目代码。它沉淀的是通用工程规范、阶段流程、模板、验收门禁和工业项目避坑经验。

Repository Map

核心入口

AGENTS.md AI 必读入口，说明不同阶段该读哪些规范。
SIMPLE_HMI.md 总规范，定义项目定位、适用范围、默认技术栈、去 Mock 红线和验收原则。
ENGINES.md 技术栈判定规则，避免 AI 静默选择后端技术方案。
docs/00_INDEX.md 阶段规范索引，串起规划、架构、Demo、开发、调试、部署、验收。
deployment_control/README.md 部署控制能力规范，用于评估版本包、健康检查、回滚和自愈是否需要接入。

How To Use

推荐给 AI 的第一条指令

请先读取 AGENTS.md、SIMPLE_HMI.md、docs/00_INDEX.md 和 ENGINES.md，
判断当前处于哪个阶段，再按对应规范和模板输出结果。
